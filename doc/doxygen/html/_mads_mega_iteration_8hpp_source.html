54 /// Class for the mega iterations of MADS.
55 /**
56 Manager for Mads iterations.
57  Steps:
58  - Generate a lot of points over multiple meshes, using different Search and Poll strategies.
59  - Evaluate points
60  - Post-processing
61 
62  \note As an hypothesis, the time load is taken by the evaluation,
63  which is parallelized over all evaluations simultaneously.
64  The iteration generation, including trial points generation,
65  has little time load, so they do not need to be parallelized.
66  It is also preferable to keep parallelization to the only place where
67  it matters the most to avoid errors.
68  There is no parallelization at the algorithmic level.
69  Algorithms are run in main thread(s) only; Secundary threads are available for evaluations.
70 */
