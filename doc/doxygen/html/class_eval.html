Detailed Description

Class for the representation of an evaluation at a point.

Note
We have a separate Eval from EvalPoint so that a Point can have multiple evaluations.

◆ dominates()

bool Eval::dominates ( const Eval &  eval,
const ComputeType &  computeType = ComputeType::STANDARD 
) const

Dominance.

Dominace as by definition 12.3 in the Book of Audet and Hare, Derivative-Free and Blackbox Optimization, https://doi.org/10.1007/978-3-319-68913-5 The feasible point x dominates the feasible point y
when f(x) < f(y). The infeasible point x dominates the infeasible point y
when f(x) <= f(y) and h(x) <= h(y), with at least one strict inequality. Otherwise, return false

Parameters
evalThe right-hand side object – IN.
computeTypeHow to compute f and h – IN
Returns
A boolean equal to true if *this dominates eval.

bool Eval::goodForCacheFile ( ) const

Should this point be saved to cache file? Based on the eval status only. These eval statuses are good: EVAL_OK, EVAL_FAILED, EVAL_USER_REJECTED, EVAL_ERROR. These eval statuses are not good: EVAL_NOT_STARTED, EVAL_IN_PROGRESS, EVAL_WAIT, EVAL_STATUS_UNDEFINED.
